项目图片资源规范

本文档规定了项目中所有图片资源的命名、尺寸、优化和组织标准，旨在确保团队协作效率和资源管理一致性。

一、图片命名规范

1. 基本原则

唯一性：确保每个文件名在项目中唯一

兼容性：文件名中不要使用特殊字符（如空格、&、%等），推荐使用下划线(_)或连字符(-)分隔单词

2. 命名格式

2.1 通用命名格式

[类型前缀]_[描述]_[属性]_[序号].png
  • 类型前缀：表示图片所属的功能类型（见2.2类型前缀表）
  • 描述：简洁描述图片内容（英文或拼音，避免生僻词汇）
  • 属性：可选，表示图片的特殊属性（如颜色、状态等）
  • 序号：必要时使用数字序号区分同类图片

2.2 类型前缀表

前缀 含义 示例
adj 形容词卡片 adj_happy_1.png
noun 名词/物品卡片 noun_sword_2.png
verb 动词卡片 verb_attack_3.png
buff 状态效果图标 buff_poison_4.png
char 角色相关图片 char_player_idle.png
ui UI元素 ui_button_ok.png
bg 背景图 bg_main_menu.png
effect 特效图片 effect_explosion_5.png
spine Spine动画图集 spine_dragon_6.png
bullet 弹道特效 bullet_fire_7.png

3. 特殊类型图片命名

3.1 角色/敌人图片

char_[角色名称]_[状态/动作].png

示例：char_hero_idle.png、char_boss_attack.png

3.2 UI元素图片

ui_[界面名称]_[元素类型]_[状态].png

示例：ui_menu_button_normal.png、ui_settings_slider_disabled.png

3.3 特效图片

[effect/bullet]_[特效名称]_[序号].png

示例：effect_fire_1.png、bullet_ice_2.png

3.4 Spine动画图集

spine_[角色/物品名称].png

示例：spine_dragon.png、spine_skeleton.png

二、图片大小与裁切规范

1. 通用原则

  • 保持适当的尺寸，避免过大影响性能
  • 确保图片比例一致，避免拉伸变形
  • 对于需要缩放的图片，保持原始清晰度

2. 具体尺寸要求

图片类型 推荐尺寸 格式 说明
卡片图片 200×300px PNG 形容词、名词、动词卡片统一尺寸
角色头像 128×128px PNG 圆形头像需预留足够边缘
UI按钮 80×80px / 120×40px PNG 根据界面布局调整
背景图 1920×1080px PNG 适配游戏分辨率
特效图片 不超过512×512px PNG 保持特效清晰同时控制文件大小
弹道图片 64×64px / 128×32px PNG 根据弹道类型调整
Spine图集 不超过2048×2048px PNG 合理打包以减少Draw Call

3. 图片优化要求

  • 所有PNG图片需进行压缩，推荐使用TinyPNG等工具
  • 对于色彩丰富的图片，使用RGB模式；对于简单图形，使用索引颜色模式
  • 移除图片中不必要的透明区域，减少文件大小
  • 对于序列帧动画，确保帧之间的尺寸一致

三、资源组织规范

1. 目录结构

Assets/
├── Art/
│   ├── Characters/    # 角色图片
│   ├── UI/            # UI元素
│   ├── Effects/       # 特效图片
│   ├── Backgrounds/   # 背景图
│   └── Spine/         # Spine动画资源
├── Resources/
│   ├── WordImage/     # 词汇相关图片
│   │   ├── Adj/       # 形容词卡片
│   │   ├── Noun/      # 名词卡片
│   │   ├── Verb/      # 动词卡片
│   │   └── Buffs/     # 状态效果图标
│   └── VFX/           # 特效资源
└── Prefabs/
    └── UI/            # UI预制件

2. 版本管理

版本标识

同一图片的不同版本使用版本号标识，如ui_button_v1.png

及时清理

过时的图片资源应及时清理，避免占用空间

源文件备份

重要图片资源的源文件应备份在项目外的存储位置

四、示例

1. 正确命名示例

adj_happy_1.png noun_sword_fire_2.png verb_heal_3.png buff_poison_4.png char_dragon_attack.png ui_menu_button_normal.png

2. 不规范命名示例

图片1.png 过于模糊，无法识别内容
sword_final_final.png 重复冗余词汇
fire bullet_001.png 包含空格，可能导致兼容性问题
技能_图标_1.png 中文命名，推荐使用英文或拼音
